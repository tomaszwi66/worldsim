## What is WorldSim?

WorldSim lets you build a **playable neural world model** from scratch using footage you record yourself вҖ” no game engine, no simulation environment. Just your phone camera and a keyboard.

The model learns to *imagine* what the next video frame should look like based on:
- The current frame (visual observation)
- Your keyboard input (W/A/S/D action)

Once trained, every frame you see during gameplay is **100% generated by the neural network** вҖ” pure learned imagination.

The architecture is a **Convolutional RSSM** (Recurrent State Space Model), inspired by [DreamerV3](https://arxiv.org/abs/2301.04104).

## Step 1 вҖ” Collect Training Data

Open the data collector app in your browser (works on phone and desktop):

рҹ‘ү **[https://websim.com/@TomaszW/worldmodel-data-collector](https://websim.com/@TomaszW/worldmodel-data-collector)**

The app is built on [WebSim](https://websim.com) вҖ” a platform for creating and sharing interactive browser apps. No installation needed, just open the link.

**How to record:**
1. Open the link on your phone or desktop
2. Move the camera around your environment
3. Press W / A / S / D on your keyboard (or on-screen buttons) to log movement direction
4. The app captures frames at 15 FPS and records the action vector `[dx, dy]` for each frame
5. When done, tap **Export** вҖ” you'll get a ZIP with:
   - `frame_00000.jpg`, `frame_00001.jpg`, ... (video frames)
   - `dataset.jsonl` (one JSON record per frame: timestamp, action, accelerometer, gyroscope, optical flow)
   - `metadata.json` (recording config)

**Tips for better data:**
- Record at least 500вҖ“1000 frames for reasonable quality
- Cover all directions вҖ” spend equal time moving W, S, A, D
- Keep movements smooth and continuous
- Record multiple scenes for better generalization

**Data format (`dataset.jsonl`):**
```json
{"t": 13588, "action": [0, 1], "frame": "frame_00000.jpg", "flow": [0.09, 0.12], "accel": [...], "gyro": [...]}
```

Action vector: `[dx, dy]` where each value is `-1`, `0`, or `1` вҶ’ 9 possible actions total.

## Step 2 вҖ” Upload to Kaggle

1. Go to [kaggle.com/datasets](https://www.kaggle.com/datasets) вҶ’ **New Dataset**
2. Upload all `frame_XXXXX.jpg` files + `dataset.jsonl` + `metadata.json` into the same flat folder
3. Name your dataset (e.g. `my-worldsim-data`) and publish it

---

## Step 3 вҖ” Train on Kaggle

1. Go to [kaggle.com/code](https://www.kaggle.com/code) вҶ’ **New Notebook**
2. Upload `notebooks/train.ipynb` from this repo
3. Add your dataset as input
4. Edit the path at the top of the notebook:

```python
BASE = Path('/kaggle/input/datasets/<your-username>/<your-dataset-name>')
```

5. Enable **GPU accelerator** (T4 Г— 2 is free) вҶ’ **Run All**

Training runs for 100 epochs (~30вҖ“60 min on T4). When done, the notebook saves:
- `worldmodel_v2.pt` вҖ” trained model weights
- `trajectory.gif` вҖ” sample imagined trajectory

Download both from the **Output** tab.

## Step 4 вҖ” Play Locally

Run:


## Model Architecture

```
 frame_t  +  action_t
      в”Ӯ            в”Ӯ
 CNN Encoder    Action Embedding
      в”Ӯ            в”Ӯ
  z_posterior в”Җв”Җв”Җв”Җв”ҳ
      в”Ӯ
  GRU Cell  в”Җв”ҖвҶ’  h_{t+1}  (world state memory)
      в”Ӯ
  CNN Decoder
      в”Ӯ
  frame_{t+1}  (predicted next frame)
```

**Key components:**

| Component | Details |
|-----------|---------|
| CNN Encoder | 5Г— Conv2d, stride 2, ResBlocks, 128вҶ’latent 256 |
| CNN Decoder | 5Г— ConvTranspose2d, ResBlocks, latentвҶ’128Г—128 |
| Residual Blocks | Conv + GroupNorm + SiLU |
| GRU | hidden_dim = 512 |
| Latent | VAE-style, dim = 256 |
| Action | Embedding, 9 classes вҶ’ dim 64 |

**Loss function:**
```
L = MSE(pred, target)
  + 0.1 Г— VGG16_perceptual(pred, target)
  + 0.01 Г— KL(posterior вҖ– prior)
```

The **perceptual loss** (VGG16 features) is critical for sharp, realistic frame prediction. Pure MSE produces blurry results.

## Known Limitations

- **Small datasets** вҶ’ blurry or unstable frames, especially for rare actions
- **A/D drift** вҖ” if training data contains fewer lateral movements than forward/backward, the model performs worse on A/D. Fix: record more balanced data
- **Long-term drift** вҖ” after many steps without a real frame anchor, the model may drift. Press `R` to reset
- **Scene-specific** вҖ” model trained on one scene doesn't generalize to unseen environments
